# AI饮食管家 (Meal Planner)

> 支持 **DeepSeek · 豆包 · 通义千问** 等任意 AI 模型，安装即用，无需 API Key。

## 一句话描述

你录入附近的店和好吃的菜，AI明天帮你选 —— 不是随机抽，而是看过你吃过什么、爱吃什么、该补什么之后，给你一个有理由的推荐。

## 为什么必须有AI

传统做法无非三种：随机抽一个 (没意义)、按规则筛 (规则永远写不完)、自己翻记录 (太累)。AI的价值在于：

| 维度 | 传统做法 | AI做法 |
|------|----------|--------|
| 口味偏好 | 固定规则（if辣 then推荐川菜） | 从历史记录中学习你实际选择模式 |
| 营养均衡 | 人工录入每道菜的热量蛋白，不现实 | 只需输入菜名，AI预估营养结构并评估均衡性 |
| 避免重复 | 硬规则排除最近N天吃过的 | AI考虑"多久没吃"和"吃了会不会腻" |
| 推荐理由 | 无 | 每条推荐附带可理解的决策理由 |
| 约束冲突 | 死路（无选项则报错） | AI在冲突约束间做权衡，给出优先推荐 |

**一句话**：没有AI，这功能根本做不出来——要么随机到无效，要么规则写到崩溃。

## 前置依赖

- Python 3.9+（脚本用）
- 安装到 WorkBuddy 后，AI 推理由 WorkBuddy 内置模型（DeepSeek）直接完成，**无需额外 API**

## 使用方法

### 第一步：设置你的画像

```bash
python skill/scripts/meal_planner.py profile set
```

交互式输入：身高、体重、目标（减脂/增肌/维持）、预算上限、忌口、口味偏好。

### 第二步：录入店铺和菜品

```bash
python skill/scripts/meal_planner.py shop add   # 添加店铺
python skill/scripts/meal_planner.py dish add   # 为店铺添加菜品
```

输入：店名、菜名、价格、你觉得好吃的程度(1-5星)。

### 第三步：记录每天吃了什么

```bash
python skill/scripts/meal_planner.py log
```

交互式记录今日三餐（选店选菜），AI评估营养并给出反馈。

### 第四步：获取明日推荐

```bash
python skill/scripts/meal_planner.py recommend
```

AI综合以下因素给出推荐：
- 你的偏好画像
- 过去7天吃了什么（避免重复、补足缺失营养）
- 各店铺好吃菜的评分
- 预算限制
- 可选：天气、场合、心情

输出示例：
```
[明日推荐]
  早餐: 食堂 - 豆浆 + 鸡蛋灌饼 (¥6)
  理由: 你今天早餐没吃，明早补碳水和蛋白质，帮你撑到中午
  午餐: 粤味轩 - 白切鸡饭 + 炒时蔬 (¥18)
  理由: 你3天没吃白切鸡了(你给了5星)，配蔬菜均衡一下
  晚餐: 兰州拉面 - 牛肉拉面 小份 (¥12)
  理由: 今晚热量已达上限，明天晚饭控制一下，小份够了你晚上一般吃不多
  全天消费: ¥36 (预算内 ✓)
  营养评估: 碳水偏多，建议早餐加个鸡蛋
```

## 工作流程

```
1. 用户录入店铺+菜品 (一次性)
      ↓
2. 用户设置画像 (一次性)
      ↓
3. 每天记录吃了什么
      ↓         ↓
4. AI评估当日营养 ←→ 更新历史记录
      ↓
5. AI生成明日推荐 (综合画像+历史+偏好+预算)
      ↓
6. 用户反馈好吃/不好吃 → AI学习调整
```

## 数据存储

所有数据存为JSON文件于 `data/` 目录：
- `profile.json` — 用户画像
- `shops.json` — 店铺列表
- `dishes.json` — 菜品库
- `history.json` — 饮食历史记录
